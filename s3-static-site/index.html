Routes

---- IP Address ----
https://ipv4.rawrify.com/ip - your public IPv4 address
https://ipv6.rawrify.com/ip - your public IPv6 address

---- User Agent ----
https://user-agent.rawrify.com/user-agent - your user agent

---- Base64 Encoding/Decoding ----
https://www.rawrify.com/base64 - base64 encoding and decoding - refer to wiki for how to use

---- Weather ----
https://www.rawrify.com/temperature?latitude={latitude}&longitude={longitude} - refer to wiki for more info

---- Symmetric Encryption ----
https://www.rawrify.com/encryption - refer to wiki for how to use
https://www.rawrify.com/decryption - refer to wiki for how to use

----- Asymmetric Encryption -----
https://www.rawrify.com/asymmetric-encrypt - refer to wiki for how to use
https://www.rawrify.com/asymmetric-decrypt - refer to wiki for how to use

---- Location ----
https://www.rawrify.com/location?country - the country code that your public IP address is associated with (adherent to ISO-3166-1 alpha-2)
https://www.rawrify.com/location?city - the city that your public IP address is associated with
https://www.rawrify.com/location?country-name - the name of the country that your public IP address is associated with
https://www.rawrify.com/location?coordinates - the latitude and longitude coordinates associated with your public IP address
https://www.rawrify.com/location?full - all of the above information delivered within a JSON object

----- Steganography -----
- Please note these have strict size limitations since this is a free service. If you want/need larger file support, you are free to fork Rawrify into your own environment
https://www.rawrify.com/steg-embed - refer to wiki for how to use
https://www.rawrify.com/steg-retrieve - refer to wiki for how to use

----- Time Functionality -----
https://www.rawrify.com/utc-time-now - Get current time in UTC
https://www.rawrify.com/epoch-now - Get current time in Unix epoch
https://www.rawrify.com/get-timezone-time?timezone=TARGET_TIMEZONE - Get current time in a specific timezone - refer to this link and use the "TZ Database Name" column
https://www.rawrify.com/compare-timezones?timezone1=TARGET_TIMEZONE1&timezone2=TARGET_TIMEZONE2 - Compare two timezones for difference in hours - refer to above link for timezones

What is this?

An ultra-lightweight, easy-to-use, free, and open-source utility API for retrieving things like a client's IP address, user-agent, quick base64 encoding/decoding, and so on. Refer to routes and the documentation for more info on what Rawrify can do.

How can I learn to use this?

Refer to the official documentation for in-depth examples of and explanations of how to use Rawrify.

Why would I need a tool like this?

Programmatic access to information like IP address, user agent, base64 data, and so on serves a variety of purposes.

- For example, say your program executes on a remote host and needs to grab the host's IP address. Just query Rawrify for the public IP and you're good to go.
- Another example, say you're building infrastructure using Terraform and want to dynamically assign your own public IP address to a security group (a real use-case I run into frequently). To avoid hardcoding an IP address, you can create an "http" data source with a "url" argument pointing to https://ipv4.rawrify.com/ip to grab your public IP! Much more clean and portable.
- Say you don't want to/don't care to implement base64 encoding in your chosen language/program. Bam. Here you go. Quick and easy encoding/decoding.

Why did you make this?

I'm a huge fan of extremely lightweight services that deliver only what you need, nothing else. Don't get me wrong-- I can appreciate over the top, insanely feature rich web apps (RIP Flash game sites), but these lightweight tools hold a special place in my personal and professional heart. Other services like icanhazip.com were a huge inspiration for this.

If my first cellphone could open this site, it's something special.

Do you collect/store my data?

No. I do not collect nor store any client-specific information on my end. This is subject to change if required by law in the future, but I have no plans/desire to implement such tracking as of now. Refer to the privacy policy for more information.
